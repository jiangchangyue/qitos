# v0.7 Handoff Scope

## What was in v0.6 scope (DelegateTool only) — COMPLETE

- `DelegateTool` with nested `Engine.run()` — synchronous delegation
- `AgentSpec` with `model_override`, `tools_override` fields
- `AgentRegistry.get_delegate_tools()` and `get_handoff_tools()`
- `DelegateEventInterceptor` for first-class DELEGATE_START/END events
- Sub-trace writer with depth-aware run_id
- Depth guard: `MAX_DELEGATE_DEPTH=3`

## What was deferred to v0.7 — NOW COMPLETE

1. **Decision.handoff() Engine loop modification** — DONE
   - Full context strategy implementation (FULL/SUMMARY/ISOLATED) for handoff
   - Shared memory injection on handoff
   - State adapter invocation during agent switching

2. **HandoffTool as first-class tool** — DONE
   - HandoffTool wired into Engine action pipeline via `_check_handoff_from_tool_result()`
   - Engine intercepts `transfer_to_*` tool calls and converts to `Decision.handoff()`
   - Enriched parameters: `message` and `memory_keys`

3. **Canonical multi-agent templates** — DONE
   - Manager-Worker (`examples/patterns/manager_worker_handoff.py`)
   - Planner-Executor (`examples/patterns/planner_executor_handoff.py`)
   - Proposer-Verifier (`examples/patterns/proposer_verifier_handoff.py`)

4. **qita handoff timeline** — DONE
   - Context strategy labels on handoff arrows
   - Per-agent token/cost breakdown below gantt chart
   - Click-to-expand handoff detail panels

5. **Async delegate** — DEFERRED to v0.8
   - Requires deeper event loop integration

## Key design decisions made in v0.7

- `_execute_handoff_step()` extracted as a reusable Engine method, called from both `Decision.handoff()` and HandoffTool interception
- `_check_handoff_from_tool_result()` handles two detection paths: (1) `_action_runtime` returns Decision directly, (2) HandoffTool.execute() returns a dict with `handoff_target`
- SharedMemoryManager uses explicit `grant_read_access()` for cross-namespace reads rather than automatic namespace exposure
- `HandoffContext.payload` written to target's shared memory namespace (prefixed `handoff_payload:`) rather than injected into state
- State adaptation uses `setattr()` instead of `__dict__.update()` to avoid leaking private attributes
- `AgentRegistry.validate_topology()` returns warnings (not errors) — cycles and isolated agents are informational

## What is deferred to v0.8

1. **Async delegate** — Support for asynchronous delegation with event loop integration
2. **Handoff graph visualization** — Full graph rendering in qita (beyond the gantt chart)
3. **Sub-agent env sharing** — Formal env sharing protocols during handoff
4. **Handoff checkpointing** — Save/restore handoff state for long-running multi-agent runs
