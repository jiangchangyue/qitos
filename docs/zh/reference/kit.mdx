`qitos.kit` 提供可复用的智能体编写积木。所有组件都挂接在 `AgentModule + Engine` 主流水线上，不会引入第二个编排器。

```python
from qitos.kit import ReActTextParser, WindowMemory, CodingToolSet, ...
```

如果你正在做 v0.4 的多模型研究，推荐优先查看 `qitos.harness`：

```python
from qitos.harness import build_harness_policy, build_model_for_preset
```

`OpenAICompatibleModel(...)` 仍然适合手工接单一路径；但如果你想让同一个智能体在多个模型家族之间切换，家族预设才是推荐入口。

## 如何阅读这页

把这页看成 QitOS 编写的能力地图：

| 如果你在决定…… | 优先查看 |
|---|---|
| 模型该如何输出动作 | 解析器 |
| 长时运行如何保住有效上下文 | 记忆与历史 / 上下文压缩 |
| 要暴露哪些工具 | 工具集 |
| 从哪些现成注册表开始 | 预设构建器 |
| 如何加规划或搜索结构 | 规划 |

## 能力总览

### 解析器

- `ReActTextParser`：解析 `Thought:` / `Action:` 文本协议
- `JsonDecisionParser`：解析 JSON 决策对象
- `XmlDecisionParser`：解析 XML 协议
- `MiniMaxToolCallParser`：处理模型原生工具调用结构
- `TerminusJsonParser` / `TerminusXmlParser`：处理带显式终止信号的格式

### 工具预设

- `coding_tools(...)`：标准编码工作区
- `advanced_coding_tools(...)`：更偏 Claude 风格的编码预设
- `web_tools()`：网页研究与提取
- `task_tools(...)`：持久任务板工作流
- `qitos.kit.tool.experimental.security_research`：显式 opt-in 的安全研究工作流
- `thinking_tools()`：显式思考记录流程

### 长时上下文能力

- `WindowHistory`
- `TokenBudgetSummaryHistory`
- `CompactHistory`
- `WindowMemory`
- `SummaryMemory`
- `VectorMemory`
- `MarkdownFileMemory`

### 规划与搜索

- `NumberedPlanBuilder`
- `PlanCursor`
- `DynamicTreeSearch`

<Tabs>
<Tab title="解析器">

解析器负责把原始大模型输出转换成决策（智能体每步的结构化决策）。最重要的原则是：**解析器必须与提示词请求的输出格式一一对应。**

<AccordionGroup>
<Accordion title="ReActTextParser">

用于解析带 `Thought:`、`Action:`、`Final Answer:` 标记的 ReAct 文本。

```python
from qitos.kit import ReActTextParser
```

当你的提示词要求模型输出文本 ReAct 协议时，就使用它。

</Accordion>

<Accordion title="JsonDecisionParser">

用于解析 JSON 输出。适合需要更严格结构化契约的场景。

```python
from qitos.kit import JsonDecisionParser
```

</Accordion>

<Accordion title="XmlDecisionParser">

用于解析 XML 输出。适合需要 XML 风格结构控制的模型或协议。

```python
from qitos.kit import XmlDecisionParser
```

</Accordion>

<Accordion title="Terminus 解析器">

`TerminusJsonParser` 与 `TerminusXmlParser` 用于带显式终止标记的协议。适合终端控制、长时交互等更强约束场景。

</Accordion>
</AccordionGroup>

</Tab>

<Tab title="历史与上下文压缩">

QitOS 把长时上下文管理当成一等能力，而不是隐形中间件。

- `WindowHistory`：最简单的最近窗口
- `TokenBudgetSummaryHistory`：超 token 预算时自动摘要
- `CompactHistory`：多阶段上下文压缩，支持警告、轮次分组、摘要压缩与可观测元数据

如果你在研究长时运行智能体，`CompactHistory` 往往是最值得重点理解的组件。

</Tab>

<Tab title="记忆">

记忆适配器负责把跨步信息存起来，并在需要时重新检索：

<AccordionGroup>
<Accordion title="WindowMemory">

保留最近 N 条记录。

</Accordion>

<Accordion title="SummaryMemory">

保留最近记录，并把更早记录压缩成摘要。

</Accordion>

<Accordion title="VectorMemory">

按语义相似度检索历史记录，适合长时研究与语义召回。

</Accordion>

<Accordion title="MarkdownFileMemory">

把记忆写入本地 Markdown 文件，适合持久化与人工审阅。

</Accordion>
</AccordionGroup>

</Tab>

<Tab title="工具集">

QitOS 的工具集帮你直接组合出一个工作环境，而不是一组松散工具。

- `CodingToolSet`：文件、编辑、终端、任务、笔记本等编码环境
- `TaskToolSet`：待办与进度跟踪
- `SecurityAuditToolSet`：代码与配置安全审计
- 其他更窄场景的捆绑包：`report_tools(...)`、`notebook_tools(...)`、`epub_tools(...)`

一般建议：

- 学习框架时手工构建注册表
- 做真实工作流时切换到预设工具集

</Tab>

<Tab title="规划">

规划辅助工具帮你在不重写运行时的情况下引入显式计划与搜索结构：

- `NumberedPlanBuilder`：从编号列表构建计划
- `PlanCursor`：记录计划执行进度
- `DynamicTreeSearch`：在分支候选之间做选择与扩展

</Tab>
</Tabs>
