本页会带你从零开始，搭建一个模型驱动的最小编程智能体（coding agent），直到可视化查看追踪记录（trace），全程大约 2 分钟。

QitOS 运行时（runtime）的实际效果如下：

![QitOS 运行时快照](/images/qitos-cli.png)

<Note>
如果你还没有远程模型服务和 API 密钥，建议先阅读 [前置准备](/zh/prerequisites)。其中说明了去哪里注册、如何创建 API 密钥，以及如何验证兼容 OpenAI 的服务商（provider）是否可用。
</Note>

## 前置条件

- Python 3.10 或更高版本
- `pip`
- 一个兼容 OpenAI 的 API 密钥

## 操作步骤

<Steps>
  <Step title="安装 QitOS">
    ```bash
    pip install "qitos[models]"
    ```
  </Step>

  <Step title="设置服务商配置">
    ```bash
    export OPENAI_API_KEY="sk-..."
    export OPENAI_BASE_URL="https://api.siliconflow.cn/v1/"
    export QITOS_MODEL="Qwen/Qwen3-8B"
    ```

    <Tip>
      任意兼容 OpenAI 的接口地址都可以使用。如果你不是直接使用 OpenAI，请将 `OPENAI_BASE_URL` 指向对应的服务商。
    </Tip>
  </Step>

  <Step title="运行最小编程智能体">
    ```bash
    qit demo minimal
    ```

    这条命令会先创建一个包含最小缺陷的工作区（workspace），配置真实模型，让最小编程智能体修复缺陷、执行验证，并输出 qita 可识别的追踪记录。你会看到类似输出：

    ```
    model_name: Qwen/Qwen3-8B
    workspace: .../playground/minimal_coding_agent
    target_file: buggy_module.py
    trace_run: .../runs/qitos_minimal_coding_<timestamp>
    final_result: Patch applied and verification passed.
    stop_reason: final
    next_step: qita board --logdir runs
    ```
  </Step>

  <Step title="用 qita 查看运行轨迹">
    ```bash
    qita board --logdir runs
    ```

    在浏览器中打开 [http://127.0.0.1:8765](http://127.0.0.1:8765)。面板（board）会列出日志目录中的所有运行记录，包括步骤数、停止原因和最终结果。点击任意运行记录的 **view** 查看逐步轨迹，或点击 **replay** 按钮进行回放（replay）。

    <Tip>
      打包好的演示与 [`examples/quickstart/minimal_agent.py`](https://github.com/Qitor/qitos/blob/main/examples/quickstart/minimal_agent.py) 走的是同一条最小编程智能体路径。想阅读或修改代码，可以从这个示例文件入手。
    </Tip>
  </Step>
</Steps>
